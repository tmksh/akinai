基礎知識

そもそも「API」ってなに？

連携を始める前に、まずは基本的な仕組みを理解しましょう。

レストランで例えると...

APIは「レストランの注文窓口」のようなものです。

あなた（お客さん）は、メニューを見て「カレーをください」と注文します。
キッチン（akinaiのサーバー）が料理を作って、できた料理（データ）を渡してくれます。

つまり API = あなたのサイトが akinai にデータを「注文」するための窓口 です。
あなたのサイトが「商品一覧をください」とリクエストすると、akinai が商品データを返してくれます。

専門用語が出てきたら

このドキュメントでは、 APIApplication Programming Interface の略。プログラム同士がデータをやり取りするための「窓口」のこと。 や エンドポイントAPIのURL（住所）のこと。例: /api/v1/products などの用語に点線がついています。マウスを乗せると意味が表示されます。

JSON（ジェイソン）とは？

プログラムがデータをやり取りするときに使うフォーマットです。例えば商品データは { "name": "Tシャツ", "price": 3000 } のように、名前と値のペアで表現されます。

まずは体験

こんなに簡単にデータが取れます

たった数行のコードで商品データを取得できます。

JavaScript 
// akinai から商品一覧を取得する例
const response = await fetch('https://yourshop.akinai.jp/api/v1/products', {
  headers: {
    'Authorization': `Bearer ${process.env.AKINAI_API_KEY}`
  }
});

const data = await response.json();

// data.products に商品の配列が入っている
console.log(data.products);
// → [{ name: "Tシャツ", price: 3000, ... }, ...]
yourshop の部分は置き換えてください

URLの yourshop の部分は、あなたの akinai ショップのサブドメインに置き換えてください。

API一覧

使えるAPIの一覧

akinai が提供するすべてのAPIです。詳しい使い方は各リンク先で確認できます。

メソッド エンドポイント 何ができるか
GET /api/v1/products 商品の一覧を取得する（検索・絞り込みもできる）
GET /api/v1/products/:id ひとつの商品の詳細を取得する
GET /api/v1/categories カテゴリの一覧を取得する
GET /api/v1/contents ニュース・ブログ記事の一覧を取得する
GET /api/v1/contents/:id ひとつの記事の全文を取得する
POST /api/v1/cart/validate カートの中身の在庫確認・金額計算をする
GET /api/v1/shipping-options 送料の選択肢を取得する
POST /api/v1/checkout クレジットカード決済ページを作成する
POST /api/v1/orders 注文を作成する（Stripe以外の決済用）
GET /api/v1/customers 顧客の一覧を取得する
GET と POST の違い

GET = データを「もらう」（取得する）ときに使います。
POST = データを「送る」（作成・処理する）ときに使います。
